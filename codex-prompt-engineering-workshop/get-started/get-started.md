# Get Started

## Introduction

In this lab, you will prepare the working environment for the Codex prompt engineering workshop. The goal is to confirm that you can access the workshop source material, use the Oracle style assets appropriately, and enter the hands-on labs with a clear understanding of the expected tools and outputs.

Estimated Time: 10 minutes

### Objectives

In this lab, you will:

* Confirm access to the workshop content and supporting source material
* Review the tools and accounts required for the hands-on flow
* Establish the working assumptions that later labs will build on

### Prerequisites

This lab assumes you have:

* Access to Codex or a comparable Codex-enabled environment
* Access to Oracle documentation, Oracle blogs, and Oracle brand assets
* Familiarity with basic OCI concepts such as compartments, networking, and identity
* A target use case for a workshop, demo, or internal enablement asset

## Task 1: Confirm the source material

1. Review the workshop source categories you plan to use throughout this lab series:

    * Oracle documentation
    * Oracle blogs
    * Oracle Style Guide assets
    * Any approved internal notes or workshop inputs

2. Verify that you can access each source before you begin generating content.

    > **NOTE:** This workshop assumes source-grounded output. If a source is unavailable, adjust the lab scope before proceeding rather than asking Codex to infer missing facts.

3. Decide which source category will serve as the primary authority for technical accuracy.

## Task 2: Confirm the toolset

1. Verify that your working environment supports the activities used in this workshop:

    * Markdown authoring
    * File inspection and editing
    * Browser-based source review
    * Optional presentation or document generation

2. Confirm whether your work will be performed primarily in a desktop environment, a local repository workflow, or a shared workspace.

3. Record any constraints that should influence later prompts, such as approval requirements, limited tool access, or environment-specific branding rules.

    > **TIP:** If your environment has strict permissions or limited tooling, mention that explicitly in your setup prompt so Codex can avoid suggesting steps that will fail in practice.

## Task 3: Set workshop expectations

1. Review the intended outputs of this workshop:

    * A Codex configuration approach
    * A grounded presentation draft
    * A LiveLabs-style lab draft
    * An OCI deployment plan
    * An APEX buildout plan
    * A pattern for extending an existing workshop

2. Decide whether your immediate goal is:

    * Workshop planning
    * Asset generation
    * Lab conversion
    * LiveLabs production hardening

3. Write down the success criteria you will use to evaluate the later labs.

4. You may now [proceed to the next lab](#next).
