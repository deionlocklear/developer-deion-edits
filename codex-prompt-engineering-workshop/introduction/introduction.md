# Introduction

## About this Workshop

This workshop is designed as a practical guide to prompt engineering with Codex for Oracle technical content creation. Rather than treating prompt engineering as a purely abstract exercise, the workshop uses a real delivery path: configure Codex, connect MCP tools, gather source material, generate a presentation, convert that presentation into a lab, deploy supporting OCI resources, build a lightweight APEX experience, and finally extend an existing workshop with a new lab.

The intent is to help technical sellers, solution engineers, developer advocates, and workshop builders develop repeatable prompting habits that produce useful artifacts instead of one-off demos. The focus is on clear instructions, scoped tasks, source grounding, review loops, and reliable output formats.

Estimated Workshop Time: 2 hours 30 minutes

### Objectives

In this workshop you will:

* Configure Codex for practical workshop-development tasks
* Understand how MCP expands Codex with additional tools and context sources
* Use blogs, Oracle documentation, and the Oracle Style Guide to build a presentation
* Convert presentation content into a structured hands-on lab
* Define the OCI resources required to support a realistic workshop environment
* Plan an APEX buildout that complements the lab narrative
* Extend an existing workshop with a new lab using consistent structure and prompting patterns

### Prerequisites

This lab assumes you have:

* Access to Codex or a comparable Codex-enabled environment
* Access to Oracle documentation, Oracle blogs, and Oracle brand assets
* Familiarity with basic OCI concepts such as compartments, networking, and identity
* A target use case for a workshop, demo, or internal enablement asset

This first-pass version is intentionally workflow-oriented. It does not assume every environment is already provisioned, and it is designed to help you shape the workshop assets before you finalize tenancy-specific implementation details.
